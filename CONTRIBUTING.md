# Contributing Guidelines

We welcome contributions from everyone! To keep things clear and positive, please follow these basic guidelines:

	1.	Be respectful and constructive — help make this a safe and friendly space for collaboration.
	2.	Open an issue first before starting major changes, to discuss the direction.
	3.	Keep your pull requests focused — one feature or fix at a time.
	4.	Write clear commit messages and provide a short description of what your PR does.
	5.	Test your changes before submitting — especially if they affect the build or sync logic.
	6.	Use English in code and documentation to make the project accessible for everyone.
	7.	Feel free to ask questions — every suggestion or idea is welcome!
	8.	If AI tools (e.g. ChatGPT, Copilot) were used, please mention this in the commit message or PR description. Transparency helps us learn and collaborate better.

Let’s build something fast and useful — together!
